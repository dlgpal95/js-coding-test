### :heavy_check_mark: 내가 푼 답안

```javascript
function solution(s) {
  if ((s.length != 4 && s.length != 6) || isNaN(parseInt(s))) return false;
  return true;
}
```

어느 부분에서 실패가 떠서 이유를 찾아보니 지수형태일때는 NaN로 반환을 안하는거였다..  
만약 `const a = '1e22'` 일때 `parseInt(a)`를 하면 `1`이 반환된다 ㅠㅠㅠ

```javascript
function solution(s) {
  if ((s.length != 4 && s.length != 6) || String(Number(s)) != String(s))
    return false;

  return true;
}
```

그래서 만약 지수형태인경우 `String(Number(s))`를 하면 `1e+22`가 되기때문에 `String(s)`인 `1e22`랑 달라서 조건에 걸리게 했다.

### :heavy_check_mark: 참고하면 좋은 다른 답안

Thanks for https://programmers.co.kr/learn/courses/30/lessons/12918/solution_groups?language=javascript

```javascript
function solution(s) {
  var regex = /^\d{6}$|^\d{4}$/;
  return regex.test(s);
}
```

정규식을 이용한 코드이다.
